# Study-design modules

Select the closest module after reading the methods, not from the title alone. A paper may use more than one module.

| Module | Extract first | Appraise first |
|---|---|---|
| Controlled experiment | allocation, intervention/exposure, comparator, independent unit, replication, blinding, outcomes | randomization, baseline balance, missing data, selective reporting, unit-of-analysis error |
| Observational study | population, sampling, exposure/predictor, outcome, timing, covariates | selection, measurement, confounding, reverse causation, attrition |
| Randomized or non-randomized clinical study | eligibility, intervention, comparator, outcomes, follow-up, harms | allocation, deviations, missing outcomes, outcome measurement, selective reporting |
| Diagnostic/prognostic study | target population, index model/test, reference outcome, horizon, validation sample | spectrum/selection, reference standard, overfitting, calibration, external validation |
| Survey | frame, recruitment, instrument, response rate, weighting, constructs | coverage, nonresponse, question validity, common-method bias, weighting |
| Qualitative study | epistemic approach, sampling, setting, data generation, coding, themes | reflexivity, sampling rationale, analytic transparency, disconfirming cases, transferability |
| Mixed methods | quantitative strand, qualitative strand, integration point | quality of each strand, integration logic, discordant findings |
| Computational/ML benchmark | dataset, split, preprocessing, model, baseline, metric, seeds, hardware | leakage, tuning on test data, metric fit, ablation, uncertainty, external validation, reproducibility |
| Simulation/modeling | assumptions, equations, parameter sources, scenarios, sensitivity analysis | structural assumptions, calibration, identifiability, uncertainty, validation |
| Methods/instrument paper | intended use, reference method, reliability, validity, operating conditions | sample adequacy, comparator validity, repeatability, robustness, scope limits |
| Case report/case series | case selection, chronology, intervention/exposure, outcome, alternative explanations | completeness, plausibility, dechallenge/rechallenge where relevant, generalizability |
| Systematic/scoping review | question, databases, dates, search, screening, extraction, included designs | search coverage, duplicate processes, bias assessment, synthesis appropriateness, update date |
| Meta-analysis | eligibility, effect measure, dependency handling, model, heterogeneity, sensitivity analyses | study bias, publication bias, incompatible pooling, multiplicity, robustness |
| Narrative review | declared scope, source-selection logic, organizing framework | selection transparency, balance, source currency, primary-source fidelity |
| Theoretical/conceptual paper | definitions, premises, argument steps, predictions, boundary conditions | internal coherence, construct clarity, alternative explanations, empirical testability |
| Commentary/editorial/policy paper | position, supporting evidence, stakeholder assumptions, recommendation | evidence selection, conflicts, feasibility, value judgments, uncertainty |
| Guideline/consensus | panel, evidence process, recommendation strength, applicability | conflicts, evidence-to-decision method, stakeholder input, update plan |

## Routing rules

- Use the original study module for empirical claims cited by a review when the primary paper is available.
- Use the review module for claims about the review's search, inclusion, synthesis, or conclusions.
- For papers with multiple experiments, datasets, cohorts, or cases, create one study row per unit.
- For humanities or argumentative scholarship, use conceptual claims and argument appraisal; do not fabricate samples, interventions, or effect sizes.
- For legal, policy, or standards documents, distinguish normative requirements from empirical findings.
- For preprints and conference papers, record publication status without treating it as a quality score.

## Appraisal tools

Use a discipline-approved tool when the user or protocol requires one. Record its exact name and version when known. Do not reproduce a proprietary checklist from memory, invent missing criteria, or translate unlike tools into one universal score.
